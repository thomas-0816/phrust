# Cranelift JIT Counter And Report Schema

Date: 2026-06-23.

Prompt 07.CL.04 defines the machine-readable counter and report surface used by
later Cranelift addendum prompts.

## VM Counter Fields

The VM counter JSON includes these JIT fields:

| Field | Meaning |
| --- | --- |
| `jit_mode` | Requested JIT mode: `off`, `noop`, or `cranelift`. |
| `jit_threshold` | Requested hot-call threshold for JIT compilation. |
| `jit_compile_attempts` | Regions submitted to the JIT tier. |
| `jit_compiled` | Regions accepted by the current JIT backend. |
| `jit_executed` | Regions that returned through the JIT path. |
| `jit_bailouts` | Regions that bailed out before or during JIT execution. |
| `jit_code_bytes` | Native code bytes generated by accepted JIT compiles. |
| `jit_compile_time_nanos` | Native compile time accumulated for accepted JIT compiles. |
| `jit_side_exits` | Structured exits from JIT code back to the interpreter. |
| `jit_side_exit_reasons` | Per-reason map for structured side exits, keyed by stable `SideExitReason` report spelling. |
| `jit_guard_failures` | Failed guards observed in JIT code or before native entry. |
| `jit_blacklisted_regions` | Regions disabled after repeated failures. |
| `jit_blacklist_reasons` | Per-reason map for process-local JIT blacklist decisions. |
| `jit_tiering_cold_functions` | Function entries kept on the interpreter because they had not reached the JIT threshold. |
| `jit_tiering_hot_functions` | Function entries admitted to JIT compilation or execution by the threshold policy. |
| `jit_tiering_eager_functions` | Function entries admitted by test-only eager mode. |
| `jit_tiering_blacklist_rejections` | Function entries rejected because the JIT region was disabled or blacklisted. |
| `jit_tiering_budget_rejections` | Function entries rejected by request compile-budget limits. |
| `jit_helper_calls` | Runtime helper calls made by JIT code. |
| `jit_fast_path_hits` | Inline JIT fast-path operations completed by native code. |
| `packed_fetch_fast_hits` | Helper-assisted packed-array int-index fetches completed by native code. |
| `packed_fetch_bounds_exits` | Packed-array fetch side exits caused by negative or out-of-bounds integer indexes. |
| `packed_fetch_layout_exits` | Packed-array fetch side exits caused by layout, element-shape, or reference guard misses. |
| `packed_foreach_sum_fast_hits` | Packed-array foreach integer reductions completed by native code. |
| `packed_foreach_sum_layout_exits` | Packed-foreach sum side exits caused by array layout, element-shape, or reference guard misses. |
| `packed_foreach_sum_overflow_exits` | Packed-foreach sum side exits caused by checked integer addition overflow. |
| `known_call_fast_hits` | Guarded known `strlen` or `count` calls completed by native code. |
| `known_call_guard_exits` | Known-call side exits caused by string or array guard misses. |
| `known_call_slow_calls` | Interpreter fallback calls after known-call guard exits. |
| `direct_call_hits` | Guarded monomorphic method calls completed by the VM/JIT dispatch helper. |
| `direct_call_fallbacks` | Generic method-call fallbacks after a direct-call miss, guard failure, or non-eligible call shape. |
| `property_load_fast_hits` | Helper-assisted monomorphic property loads completed by native code. |
| `property_load_guard_exits` | Property-load side exits caused by receiver, layout, storage, or initialization guards. |
| `property_load_layout_exits` | Property-load side exits caused by class-layout epoch mismatches. |
| `property_load_uninitialized_exits` | Property-load side exits caused by uninitialized typed properties. |
| `property_load_slow_calls` | Interpreter fallback calls after property-load guard exits. |
| `string_concat_fast_path_hits` | Guarded string/string concat helpers completed by native code. |
| `string_concat_fast_path_misses` | String-concat helper guard misses before interpreter fallback. |
| `jit_overflow_exits` | Native integer overflow exits that resumed through the interpreter. |
| `jit_slow_path_calls` | JIT slow-path or interpreter fallback calls after a guard, type, or overflow exit. |
| `jit_compile_cache_hits` | Process-local compile-cache hits. |
| `jit_compile_cache_misses` | Process-local compile-cache misses. |
| `jit_compile_cache_invalidations` | Process-local compile-cache invalidations caused by runtime layout, ABI/config, IR, or blacklist mismatch. |
| `jit_compile_descriptors` | Diagnostic list of successful Cranelift compiles with `FunctionId`, function name, IR fingerprint, code bytes, compile time, target ISA, ABI hash, and config hash. |
| `property_fetch_profiles` | Deterministic per-callsite property-fetch metadata profiles used by monomorphic property-load fast paths. |
| `method_call_profiles` | Deterministic per-callsite method-call metadata profiles used by monomorphic method-call fast paths. |

Counters are present with zero values even before later prompts start emitting
non-zero values. This keeps report parsing stable.

## JIT Stats JSON

`php-vm run --jit-stats=json` writes a compact JSON object to stderr. With
`--jit=cranelift` and the `jit-cranelift` feature enabled, the payload includes
an eligibility summary:

```json
{
  "jit": {
    "mode": "cranelift",
    "threshold": 0,
    "eligibility": {
      "considered": 2,
      "eligible": 1,
      "non_eligible": 1,
      "rejected": 1,
      "unknown": 0,
      "reports": []
    }
  }
}
```

Each report uses stable fields:

| Field | Meaning |
| --- | --- |
| `function_id` | IR function id analyzed. |
| `function_name` | Function name when available. |
| `status` | `eligible`, `rejected`, or `unknown`. |
| `candidate_kind` | Stable eligible candidate kind such as `IntLeafCandidate`, `PackedArrayFetchCandidate`, `PackedForeachIntSumCandidate`, `KnownCallCandidate`, or `StringConcatCandidate`; otherwise `null`. |
| `stats` | Function-local analysis counters. |
| `reasons` | Stable non-eligibility reason objects with `code`, `detail`, `block`, and `instruction`. |

Successful Cranelift compiles also populate `jit.compile_descriptors` in the
compact stats JSON. Optional local diagnostics such as
`just jit-cranelift-disasm` use this array to link a dump to the same
`function_id`, `ir_fingerprint`, `code_bytes`, ABI hash, config hash, and
target ISA used by the process-local compile cache. This is metadata only and
does not affect tiering, eligibility, native entry, or fallback behavior.

Prompt 07.CL.10 keeps the allowed intrinsic set empty. Calls therefore remain
non-eligible with `JIT_ELIGIBILITY_REJECT_CALL_OPCODE` until a later prompt
adds explicit helper-call lowering.

Prompt 07.CL.12 adds the first non-zero execution counters for the
constant-return native subset. In compact `--jit-stats=json`, the same metadata
is emitted as `code_bytes` and `compile_time_nanos` inside the `jit` object.

Prompt 07.CL.13 adds non-zero helper-call reporting for successful native
helper arithmetic. In compact `--jit-stats=json`, `helper_calls` records the
number of checked runtime helpers called by JIT code during the request.
Overflow and other non-zero helper statuses fall back to the interpreter and
increase `bailouts` instead of `executed`.

Prompt 07.CL.14 adds structured side-exit reporting. VM counter JSON includes
`jit_side_exit_reasons`, and compact `--jit-stats=json` includes
`side_exit_reasons`. Prompt 07.CL.17 inline integer overflow exits use this
shape:

```json
{
  "jit": {
    "side_exits": 1,
    "side_exit_reasons": {
      "overflow": 1
    },
    "overflow_exits": 1,
    "slow_path_calls": 1
  }
}
```

Actual helper status exits remain represented by `helper_status` when a later
helper-backed path returns a non-zero status through the native ABI.

Prompt 07.CL.15 adds process-local blacklist reporting. Compact
`--jit-stats=json` includes the current blacklist mode plus blacklist totals and
reasons:

```json
{
  "jit": {
    "blacklist": "on",
    "blacklisted_regions": 1,
    "blacklist_reasons": {
      "too_many_side_exits": 1
    }
  }
}
```

Stable blacklist reason spellings are `too_many_side_exits`,
`guard_failure_rate`, `compile_errors`, and `abi_mismatch`.

Prompt 07.CL.16 adds the first Cranelift big-win matrix generator. Rows emitted
by `scripts/phase7/cranelift/jit_bench_matrix.py` include the core report
counters plus `scenario` and `jit_mode` so each `--jit=off` and
`--jit=cranelift` measurement can be compared directly in
`target/phase7/cranelift/big_wins_report.json`.

Prompt 07.CL.17 adds inline checked integer add/sub/mul reporting. Successful
inline arithmetic rows report non-zero `fast_path_hits` and zero
`helper_calls`. Overflow rows report `jit_status: "side_exit"`,
`side_exit_reasons.overflow`, `overflow_exits`, and `slow_path_calls`.

Prompt 07.CL.18 reuses the same row and counter schema for simple branches and
counted loops. Executed loop and branch rows report non-zero `fast_path_hits`
and zero `helper_calls`; unsupported loop-body calls report
`jit_status: "fallback"` with `CL-GAP-LOOP-BODY-CALL`.

Prompt 07.CL.19 adds explicit numeric-loop benchmark rows for `sum_to_n`,
`fib_iterative`, `branchy_int_loop`, and repeated integer function calls.
Cranelift report rows now distinguish `compile_time_seconds`,
`execution_time_seconds`, and `total_time_seconds`; `wall_time_seconds` remains
the advisory total elapsed measurement for backward compatibility. The JSON
matrix derives execution time by subtracting known native compile time from the
measured total and clamping at zero. The same rows report side exits and native
code bytes through the stable counter bundle. There is no CI-hard speedup
threshold and no platform-specific instruction-count assumption.

Prompt 07.CL.21 adds compact stats and report counters for the first
helper-assisted packed-array int-index fetch path. The valid packed fixture
reports non-zero `fast_path_hits` and `packed_fetch_fast_hits` with exactly one
helper call. Bounds and negative-index fixtures report
`packed_fetch_bounds_exits`, while mixed-layout/reference/non-int guard misses
report `packed_fetch_layout_exits` before interpreter fallback. String-key or
untyped-index functions remain ineligible and do not raise packed-fetch fast
hits.

Prompt 07.CL.22 adds compact stats and report counters for the packed-foreach
integer sum fast path. Successful all-int packed array rows report non-zero
`packed_foreach_sum_fast_hits`, `fast_path_hits`, native `code_bytes`, and
`compile_time_nanos`, with zero helper-call counts in the per-invocation
counter. Mixed element, layout, and reference guard misses report
`side_exit_reasons.helper_status`, `packed_foreach_sum_layout_exits`, and
`slow_path_calls`. Checked addition overflow reports
`side_exit_reasons.overflow`, `packed_foreach_sum_overflow_exits`,
`overflow_exits`, and `slow_path_calls`.

Prompt 07.CL.24 adds compact stats and report counters for the conservative
string/string concat fast path. Successful typed string/string rows report
non-zero `string_concat_fast_path_hits`, `fast_path_hits`, `helper_calls`,
native `code_bytes`, and `compile_time_nanos`, with zero
`string_concat_fast_path_misses`. String/int conversion and object
`__toString` fixtures remain outside the fast path and report
`jit_status: "fallback"` without incrementing string-concat fast counters.

Prompt 07.CL.25 adds the metadata-only property-fetch profile surface. VM
counter JSON now includes `property_fetch_profiles`, a stable array keyed by
callsite. Each entry reports observed receiver `class_ids`,
`receiver_classes`, `declared_property_names`, `visibility_contexts`,
`property_slot_indexes`, `class_layout_versions`, hook and `__get` presence,
dynamic-property fallback, whether a declared visible property was seen,
uninitialized typed-property observations, the callsite state
`monomorphic|polymorphic|megamorphic`, `fast_path_eligible`, and precise
`non_eligible_reasons`. Prompt 07.CL.25 does not add a Cranelift inline
property load; it only records the metadata needed by later prompts.

Prompt 07.CL.26 adds compact stats and report counters for the conservative
monomorphic property-load fast path. Successful DTO rows report non-zero
`property_load_fast_hits`, `fast_path_hits`, `helper_calls`, native
`code_bytes`, and `compile_time_nanos`, with zero property guard exits.
Wrong-class receiver rows report `side_exit_reasons.guard_failed`,
`property_load_guard_exits`, and `property_load_slow_calls` after any earlier
matching receiver hits. Hook and magic cases remain compile-time fallbacks with
zero property-load fast counters. Uninitialized typed properties report
`side_exit_reasons.guard_failed`, `property_load_guard_exits`,
`property_load_uninitialized_exits`, and `property_load_slow_calls` before the
interpreter raises the canonical error.

Prompt 07.CL.27 adds the metadata-only method-call profile surface. VM counter
JSON now includes `method_call_profiles`, a stable array keyed by callsite.
Each entry reports observed receiver `class_ids`, `receiver_classes`,
`declaring_classes`, `method_ids`, `method_slot_indexes`,
`visibility_contexts`, `override_layout_versions`, final/private/static method
flags, public `__call` and magic fallback flags, simple positional argument
status, by-reference argument observations, callee JIT eligibility, direct
VM-call helper availability, the callsite state
`monomorphic|polymorphic|megamorphic`, `fast_path_eligible`, and precise
`non_eligible_reasons`. Prompt 07.CL.27 does not add direct native method
calls; it only records the metadata needed by later prompts.

Prompt 07.CL.28 adds the first guarded monomorphic method-call fast path for
`--jit=cranelift`. The path reuses Prompt 07.CL.27 callsite metadata and method
inline-cache targets, checks the receiver class id/name and method epoch, then
dispatches through the VM/JIT method-call helper without inlining or adding a
second native method ABI. First misses, subclass guard failures, magic
fallbacks, by-reference/named/unpacked shapes, and other non-eligible calls use
generic method lookup and increment `direct_call_fallbacks`; successful guarded
dispatches increment `direct_call_hits` and the aggregate
`jit_fast_path_hits`. The benchmark matrix includes the
`method_call_service_dto_loop` row, and the diff corpus covers repeated small
method calls, subclass fallback, magic fallback, and callee exception
propagation.

Prompt 07.CL.29 adds the conservative Cranelift tiering policy described in
`docs/adr/0786-cranelift-tiering-policy.md`. `--jit-threshold=N` is the
minimum-call threshold, `--jit-max-compile-us=N` and `--jit-max-functions=N`
bound request-local compilation, and `--jit-eager` admits first-call
compilation for tests. Compact `--jit-stats=json` reports `eager`,
`max_compile_us`, `max_functions`, `tiering_cold_functions`,
`tiering_hot_functions`, `tiering_eager_functions`,
`tiering_blacklist_rejections`, and `tiering_budget_rejections`. The benchmark
matrix records `tiering_mode` per row and includes threshold-mode hot/cold
policy rows.

Prompt 07.CL.30 adds a process-local, non-persistent compiled-function cache.
Cache keys include function id, function IR fingerprint, `JIT_RUNTIME_ABI_HASH`,
JIT config bits, and target ISA. Runtime/class layout epoch mismatches
invalidate existing entries before fallback compilation; ABI, IR, config, and
target mismatches miss by key. Reports surface `compile_cache_hits`,
`compile_cache_misses`, and `compile_cache_invalidations` in row counters, plus
the matching `jit_compile_cache_*` fields in VM counter JSON.

Prompt 07.CL.31 consolidates the Cranelift Big-Win matrix. Generated reports
use `schema_version: 2`, include a top-level `matrix` summary, and tag each row
with `matrix_family`, `matrix_family_label`, `path_kinds`, and the executed
`command`. The required matrix families are `int_leaf_calls`,
`int_counted_loop`, `packed_int_fetch`, `packed_foreach_sum`,
`known_strlen_count_call`, `string_concat`, `property_read_loop`, and
`method_call_loop`. Optional `REFERENCE_PHP` rows are advisory orientation rows;
they are not correctness gates.

Prompt 07.CL.32 consumes the schema-2 report through
`scripts/phase7/cranelift/guard_failure_report.py`. Rows therefore also carry
`side_exit_reasons` and `blacklist_reasons` maps when the VM reports them. The
guard report writes JSON and a text summary with top side exits, high-failure
rows, blacklisted candidates, and next-action recommendations.

Optional 07.CL.E can pass
`target/phase7/cranelift/polymorphic-ic/report.json` into the same analyzer with
`--experimental-ic-report`. In that mode the guard report adds an
`experimental_ic_guards` array. Each row is report-only and contains
`scenario`, `kind`, `state`, `guard_entry_count`, `max_polymorphic_entries`, and
`fallback`; megamorphic rows must show `fallback: "megamorphic_fallback"`.
Without the optional input, the array is empty and no runtime behavior changes.

Optional 07.CL.F writes `target/phase7/cranelift/framework-smoke.json` from
`scripts/phase7/cranelift/framework_smoke.py`. That report is schema-version 1
and contains `required_fixture_kinds`, `all_triggered_paths`, and per-fixture
rows with `expected_paths`, `triggered_paths`, output parity, and a compact JIT
counter summary. The report is intentionally separate from the Big-Win matrix
because its fixtures are generated local mini-app shapes rather than committed
benchmark fixtures.

## Report Types

`crates/php_perf` owns the stable Cranelift report types:

- `CRANELIFT_JIT_REPORT_SCHEMA_VERSION`
- `JitCorrectnessStatus`
- `JitExecutionStatus`
- `JitCounterSnapshot`
- `CraneliftJitReportRow`
- `CraneliftJitReport`

The generated report path remains:

```text
target/phase7/cranelift/big_wins_report.json
```

Generated reports are local artifacts and must not be committed.

## Row Fields

| Field | Meaning |
| --- | --- |
| `matrix_family` | Stable Prompt 07.CL.31 family id used by the consolidated Big-Win matrix. |
| `matrix_family_label` | Human-readable family label. |
| `target` | Stable big-win target id such as `integer_arithmetic_leaf` or `counted_loop`. |
| `fixture` | Fixture path used for the row. |
| `jit_mode` | Requested row mode: `off`, `cranelift`, or optional `reference_php`. |
| `path_kinds` | One or more path tags such as `interpreter_baseline`, `jit_helper_call_path`, `jit_fast_path`, `jit_fallback_or_skip`, `jit_side_exit_resume`, or `reference_orientation`. |
| `correctness` | JIT-off/JIT-on comparison result. |
| `jit_status` | Runtime JIT outcome for the row. |
| `command` | Normalized command used to produce the row. |
| `iterations` | Number of measured repetitions represented by the row. |
| `wall_time_seconds` | Advisory total wall-clock time in seconds. |
| `total_time_seconds` | Total elapsed timing in seconds; currently the same source measurement as `wall_time_seconds`. |
| `compile_time_seconds` | Native compile time in seconds derived from `compile_time_nanos`. |
| `execution_time_seconds` | Total time minus known native compile time, clamped to zero. |
| `instructions` | Optional instruction count when an instruction-smoke backend is available. |
| `side_exit_reasons` | Per-reason side-exit counts copied from compact JIT stats. |
| `blacklist_reasons` | Per-reason blacklist counts copied from compact JIT stats. |
| `counters` | Stable JIT counter bundle including side exits, code bytes, compile time, helper calls, and fast-path hits. |
| `vm_counters` | Additional VM-facing counters sorted by key. |
| `known_gaps` | Stable gap or skip ids relevant to this row. |

## Status Values

`correctness` values:

- `pass`
- `fail`
- `skipped`
- `orientation_match`
- `orientation_mismatch`

`jit_status` values:

- `executed`
- `fallback`
- `side_exit`
- `skipped`
- `reference`

Only rows with `correctness: "pass"` are eligible for performance comparison.
Rows with `jit_mode: "reference_php"` are advisory only.
