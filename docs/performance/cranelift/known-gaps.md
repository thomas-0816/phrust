# Performance Cranelift Known Gaps

This catalog tracks gaps specific to the Cranelift JIT path. It does
not replace `docs/performance/known-gaps.md`; it narrows the JIT-specific follow-up
items so later JIT work can close them one by one.

## Entry Format

- `Gap ID`: stable Cranelift identifier.
- `Layer`: API, feature gating, ABI, harness, native execution, benchmark
  fixtures, reports, or safety.
- `Evidence/Test`: current file, command, or missing artifact proving the gap.
- `Risk`: correctness, safety, portability, or measurement issue.
- `Planned follow-up`: work expected to close or refine the gap.
