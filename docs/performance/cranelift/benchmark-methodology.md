# Cranelift Big-Win Benchmark Methodology

Reference target: PHP 8.5.7 (`php-8.5.7`).

This methodology extends `docs/performance/methodology.md` for the
Cranelift big-wins addendum. Correctness remains the entry condition for every
measurement: a benchmark row is reportable only after the same fixture passes
with JIT disabled and enabled.

## Measurement Principles

- Run through the Nix dev shell: `nix develop -c ...`.
- Keep `--jit=off` as the baseline and source-of-truth execution path.
- Treat `--jit=cranelift` as an experiment that may skip, side-exit, or fall
  back without failing correctness. Use `--jit=noop` only for CLI/report
  plumbing checks that must not enable compilation.
- Store generated reports under `target/performance/cranelift/`.
- Do not commit generated benchmark or dump artifacts from `target/`.
- Record host, target triple, feature flags, engine commit, CLI flags,
  benchmark repetitions, and VM/JIT counters in every machine-readable report.
- Prefer instruction counts when a supported Linux Callgrind path exists; keep
  wall-clock timings advisory unless correctness and uncertainty metadata are
  present.

## Required Correctness Pair

Every fixture in the Cranelift benchmark corpus must produce equivalent
observable behavior for:

```bash
nix develop -c target/debug/php-vm run --jit=off <fixture.php>
nix develop -c target/debug/php-vm run --jit=cranelift <fixture.php>
```

Equivalence covers stdout, stderr, exit status, deterministic diagnostics, and
any generated counter status that marks the run as skipped/fallback rather than
optimized. Performance data from a row with mismatched behavior is discarded.

that stage implements the first correctness pair as:

```bash
nix develop -c just jit-cranelift-diff
```

The harness writes:

```text
target/performance/cranelift/diff.json
```

The JSON report records the engine, fixture count, optional reference PHP
sample, per-fixture `--jit=off` and `--jit=cranelift` exit codes, stdout/stderr
hashes, extracted machine-readable diagnostics, and any diff artifact paths.
that stage expands the fixture root to all of
`tests/fixtures/performance/cranelift/`. Since that stage, correctness rows
that must exercise native execution use test-only `--jit-eager`, while
threshold rows use `--jit-threshold=N` to prove hot/cold tiering behavior.

A later stage adds helper-call fixtures under
`tests/fixtures/performance/cranelift/helper-call/` and validates helper-call
counter visibility through:

```bash
nix develop -c just jit-cranelift-bench-smoke
```

The smoke writes `target/performance/cranelift/bench-smoke.json`, checks
`--jit=off`/`--jit=cranelift` parity, requires non-zero `helper_calls` for the
successful add/mul rows, and requires overflow to fall back instead of counting
as native execution.

A later stage replaces the helper-only smoke target with the first
int-arithmetic big-win matrix under
`tests/fixtures/performance/cranelift/int_arithmetic/`. The target still writes
`target/performance/cranelift/bench-smoke.json`, but the rows now cover repeated
function-call add, an arithmetic expression chain, a counted-loop row, and an
overflow correctness-only side-exit row. The full local matrix is generated by:

```bash
nix develop -c just jit-cranelift-report
```

A later stage supersedes helper-call arithmetic for eligible `int`
add/sub/mul rows. The same smoke and report targets now require non-zero
`fast_path_hits` for successful inline integer rows, require zero
`helper_calls` for those rows, and require overflow fixtures to record both
`overflow_exits` and `slow_path_calls` before interpreter fallback.

A later stage extends the matrix with simple multi-block control flow:
conditional branches over int comparisons, unconditional jumps, returns, and
counted loops whose bodies contain only eligible int operations. The smoke and
report targets now require the counted-loop and branchy rows to execute
natively with non-zero `fast_path_hits` and zero `helper_calls`. The negative
call-in-loop row remains a deterministic fallback with
`CL-GAP-LOOP-BODY-CALL`.

that stage stabilizes the numeric loop measurements by adding explicit
`sum_to_n`, `fib_iterative`, `branchy_int_loop`, and repeated integer function
call rows. The main workspace does not include `iai-callgrind`; the older
Valgrind/Callgrind smoke has been replaced by a skip-safe Callgrind gate. The Cranelift
loop matrix therefore stays JSON/wall-clock based with generous tolerance and
no hard speedup threshold. Rows now report `compile_time_nanos`,
`compile_time_seconds`, `execution_time_seconds`, `total_time_seconds`,
`side_exits`, and `counters.code_bytes` so compile cost can be evaluated
separately from execution cost.

A later stage adds the first packed-array indexed fetch row,
`packed_array_int_fetch`. Unlike inline integer arithmetic, this row is
helper-assisted by design and therefore expects exactly one helper call for the
successful native fetch. The smoke/report gates require non-zero
`packed_fetch_fast_hits` for the valid fixture and zero packed-fetch bounds or
layout exits in that row. `cranelift-guard-report` separately checks that
bounds and mixed-layout fixtures increment only their matching packed-fetch
exit counters.

A later stage adds packed-foreach integer reduction rows:
`packed_foreach_int_sum`, `packed_foreach_mixed_element`, and
`packed_foreach_overflow`. The successful row must report non-zero
`packed_foreach_sum_fast_hits`, native code bytes, compile time, execution
time, and total time. The side-exit rows must report the matching structured
reason plus either `packed_foreach_sum_layout_exits` or
`packed_foreach_sum_overflow_exits`, and must preserve `--jit=off` output and
diagnostic parity.

A later stage adds conservative string/string concat rows:
`string_concat_two_strings`, `string_concat_empty_strings`,
`string_concat_large_strings`, `string_concat_template_loop`,
`string_concat_string_int`, and `string_concat_object_to_string`. Successful
typed string/string rows are helper-backed and must report non-zero
`string_concat_fast_path_hits`, `fast_path_hits`, `helper_calls`, native code
bytes, compile time, execution time, and total time, with zero
`string_concat_fast_path_misses`. String/int conversion and object
`__toString` rows must preserve JIT-off behavior while remaining fallback rows
with zero string-concat fast-path counters.

A later stage adds property-load metadata profiles, not benchmark rows or
native property loads. The required evidence is counter JSON coverage for
monomorphic, polymorphic, megamorphic, magic `__get`, property hook, dynamic
property, and uninitialized typed-property cases plus `jit-cranelift-diff`
parity for the property-load metadata fixtures. that stage owns the first
inline property-load fast-path rows.

A later stage adds monomorphic property-load rows:
`property_load_simple_dto`, `property_load_dto_loop`,
`property_load_wrong_class_side_exit`, `property_load_hook_magic_fallback`, and
`property_load_uninitialized_error_path`. Successful DTO rows are helper-backed
and must report non-zero `property_load_fast_hits`, `fast_path_hits`,
`helper_calls`, native code bytes, compile time, execution time, and total
time. Wrong-class and uninitialized rows must preserve JIT-off behavior while
recording `guard_failed` side exits and the matching property slow-path
counters. Hook/magic rows must remain fallback rows with zero property-load
fast counters.

A later stage adds method-call metadata profiles, not benchmark rows or
direct native method calls. The required evidence is counter JSON coverage for
monomorphic receivers, stable method slots, visibility context, override/layout
versions, final/private/static flags, magic `__call` fallback status, simple
positional arguments, by-ref non-eligibility, and callee JIT/helper
availability plus `jit-cranelift-diff` parity for the method-call metadata
fixtures.

A later stage adds the first direct monomorphic method-call row,
`method_call_service_dto_loop`. The row uses
`tests/fixtures/performance/cranelift/method-call/service-dto-loop.php` and proves
the VM/JIT dispatch helper path with non-zero `direct_call_hits` plus at least
one `direct_call_fallbacks` first miss. This work item does not add inlining or a
standalone native method-call ABI, so native compile time may remain zero when
the method body is dispatched through the VM helper. There is no CI-hard
speedup threshold for the row.

A later stage adds the tiering-policy comparison rows. Existing correctness
and benchmark rows that need immediate native execution run with `--jit-eager`.
The `tiering_cold_threshold` row uses the default conservative threshold and
must report zero compiled regions plus non-zero `tiering_cold_functions`. The
`tiering_hot_threshold` row runs with a lower `--jit-threshold=N` and must
report compiled native execution plus non-zero `tiering_hot_functions`. Reports
also surface eager, blacklist, and compile-budget tiering counters.

A later stage adds process-local compiled-function handle reuse. Benchmark
rows surface `compile_cache_hits`, `compile_cache_misses`, and
`compile_cache_invalidations`; focused VM tests prove same-function reuse,
changed-IR misses, ABI mismatch misses, and runtime-layout invalidation. The
cache is not persisted to disk and must not change JIT-off/JIT-on parity.

## Big-Win Matrix

| Big-win target | First fixture class | Required counters | Completion evidence |
| --- | --- | --- | --- |
| Integer arithmetic leaf | Constant and local integer add/sub/mul functions | eligibility accepts/rejects, compile attempts, compiled regions, executed regions, fast-path hits, overflow exits, slow-path calls | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, report row |
| Counted loops | Simple integer `for`/`while` loops with stable bounds, including `sum_to_n`, `fib_iterative`, and branchy int loops | compile attempts, compiled regions, executed regions, fast-path hits, zero helper calls, compile time, execution time, total time, side exits, code bytes, blacklists | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, report row |
| Packed-array indexed fetch | Read-only packed arrays with integer indexes | array guards, bounds checks, helper-assisted fetch fast paths, `packed_fetch_fast_hits`, `packed_fetch_bounds_exits`, `packed_fetch_layout_exits`, interpreter resumes | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, guard report row |
| Packed-array foreach sum | Packed-array integer reductions | iterator guards, element guards, sum exits, `packed_foreach_sum_fast_hits`, `packed_foreach_sum_layout_exits`, `packed_foreach_sum_overflow_exits`, fallback resumes | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, report row |
| Known internal calls | Exact global `strlen($s)` and `count($a)` with one positional argument | helper-backed string/array guards, `known_call_fast_hits`, `known_call_guard_exits`, `known_call_slow_calls`, fallback resumes | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, report row |
| String concatenation | Typed string/string concat with bounded allocation plus string/int and object conversion fallbacks | helper-backed concat fast hits, concat guard misses, helper calls, compile time, execution time, fallback resumes | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, report row |
| Monomorphic property load | Stable class/property metadata loads | metadata epoch checks, property guards, helper-assisted load fast paths, wrong-class and uninitialized exits | `jit-cranelift-diff`, `cranelift-guard-report`, `jit-cranelift-bench-smoke` |
| Monomorphic method-call metadata | Stable receiver/class/method metadata profiles | receiver classes, method IDs/slots, visibility and override versions, magic/by-ref/fallback reasons | VM counter JSON tests plus `jit-cranelift-diff` |
| Monomorphic method call | Stable receiver/class/method direct calls | receiver class guards, method epoch checks, direct dispatch-helper hits, direct-call fallbacks | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, report row with `direct_call_hits` and `direct_call_fallbacks` |
| Tiering policy | Cold and hot eligible functions under threshold mode | cold/hot/eager tiering decisions, blacklist rejections, compile-budget rejections | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, report rows for `tiering_cold_threshold` and `tiering_hot_threshold` |
| Compile cache | Repeated eligible functions in one VM process | compile-cache hits, misses, invalidations; no repeated native compile for identical key | focused VM tests, `jit-cranelift-diff`, `jit-cranelift-bench-smoke` |
| Consolidated Big-Win matrix | that stage required families: int leaf calls, counted loops, packed fetch, packed foreach sum, known calls, string concat, property reads, and method calls | family coverage, interpreter baseline rows, helper-call path rows, fast-path rows, side-exit/fallback rows, optional reference orientation | `jit-cranelift-diff`, `jit-cranelift-bench-smoke`, `jit-cranelift-report` |

The matrix is intentionally narrow. Dynamic features, references, destructors,
exceptions, generators, fibers, autoload, eval, and include-sensitive behavior
must either stay outside the eligible set or side-exit before PHP-visible
behavior can diverge.

## Machine-Readable Report Contract

Cranelift benchmark reports use the `php_perf::CraneliftJitReport` schema
documented in `docs/performance/cranelift/jit-report-schema.md`. Reports are written as
JSON under:

```text
target/performance/cranelift/big_wins_report.json
```

```json

  "gate": "jit-cranelift-report",
  "status": "pass",
  "environment": {
    "engine_version": "phrust-0.0.0",
    "git_commit": "unknown",
    "rust_target_triple": "aarch64-apple-darwin",
```

Rows may report `jit_status` values of `executed`, `fallback`, `side_exit`,
`skipped`, or `reference`. Only `correctness: "pass"` rows may be used for
performance comparison. `reference_php` rows are optional orientation rows, not
gate inputs. A skipped row must include a stable reason in `known_gaps` or a
skip-reason field when the scripts add one.

`just cranelift-guard-report` consumes this Big-Win report instead of editing
fixtures. It writes `target/performance/cranelift/guard-report.json` and a text
summary with top side-exit reasons, high-failure rows, blacklisted candidates,
and `keep|specialize|blacklist|unsupported` recommendations. Optional minimizer
commands, when present, write only under `target/performance/cranelift/minimized/`.

## Validation Commands

The addendum-specific gates are added incrementally. The current standard
sequence is:

```bash
nix develop -c just jit-cranelift-diff
nix develop -c just jit-cranelift-bench-smoke
nix develop -c just jit-cranelift-report
nix develop -c just cranelift-guard-report
nix develop -c just verify-cranelift
```

All commands are intended to run inside the Nix dev shell. The shell provides
the required local tooling for these gates: Rust/Cargo, `just`, `jq`,
`python3`, `ripgrep`, and cache tooling (`sccache`/`ccache`) where available.

Before each public Cranelift just target runs feature-on work, it writes a
platform status file:

```bash
scripts/performance/cranelift/platform_check.py --out target/performance/cranelift/platform.json
```

Supported hosts record `"status": "pass"`. Unsupported hosts record
`"status": "skip"` plus a stable reason and supported-triple list; the just
target then exits successfully without generating host-specific benchmark data.
CI uploads this JSON as the machine-readable skip artifact.

The Performance CI split is:

```bash
nix develop -c just verify-performance
nix develop -c just verify-cranelift
```

The first command is default/no-JIT Cranelift coverage. The second is the
feature-on Cranelift addendum job and has no hard speedup threshold.

## Report Failure Policy

- A correctness mismatch fails the gate and blocks the next work item.
- Missing optional native-JIT support may skip with a clear reason.
- A side exit is acceptable only when interpreter resume is validated and
  counters record the exit reason.
- A benchmark row without counters is incomplete.
- Wall-clock data without environment metadata is incomplete.
- Host-specific outputs under `target/performance/cranelift/` are local or CI
  artifacts only and must not be committed.
