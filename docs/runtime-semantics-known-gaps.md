# Runtime semantics Known Gaps

This catalog imports the Runtime runtime known gaps into Runtime semantics categories.
Status values are `planned`, `known_gap`, `implemented`, or `deferred`.

Runtime semantics may replace Runtime IDs with more specific IDs as features move from
unsupported diagnostics into executable semantics. Until then, the imported IDs
remain the stable evidence trail and must stay backed by fixtures or explicit
examples.

## ID Families

```text
E_PHP_RUNTIME_REF_*
E_PHP_RUNTIME_COW_*
E_PHP_RUNTIME_ARRAY_*
E_PHP_RUNTIME_CALL_*
E_PHP_RUNTIME_OBJECT_*
E_PHP_RUNTIME_MAGIC_*
E_PHP_RUNTIME_TYPE_*
E_PHP_RUNTIME_GENERATOR_*
E_PHP_RUNTIME_FIBER_*
E_PHP_RUNTIME_INCLUDE_*
E_PHP_RUNTIME_EVAL_*
E_PHP_RUNTIME_AUTOLOAD_*
E_PHP_RUNTIME_REFLECTION_*
W_PHP_RUNTIME_GAP_*
```

## Unsupported ID Cleanup Status

Runtime semantics keeps broad Runtime IDs only when they still describe a whole deferred
capability that has no executable Runtime semantics path, such as standard-library
breadth, streams, Zend ABI, SAPI, Opcache, or JIT. Runtime paths exercised by
Runtime semantics fixtures must use category-specific IDs or one of the explicit
compatibility IDs below.

The final Runtime semantics reference-backed diff has no unclassified `fail`
rows after fixture-level classifications.

When Standard library turns a broad capability into an executable path, it must split
the broad `UNSUPPORTED` ID into narrower IDs before claiming coverage.
